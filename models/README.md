# Model Directory

This directory contains trained model checkpoints. **Model weights are not committed to Git** (see `.gitignore`). Use the instructions below to obtain them.

---

## Directory Structure

```
models/
├── baseline/               # TF-IDF + Logistic Regression baseline model
│   └── (generated by notebooks/train_colab.ipynb)
├── xlmr-base/              # Fine-tuned XLM-RoBERTa Base (main model)
│   └── (downloaded from HuggingFace or generated by training)
└── cl/                     # Continual Learning output models
    ├── CL_output_step1/    # After CL Round 1 (VLSP Part 1 + ViHSD rehearsal)
    │   ├── model/          # Model weights (safetensors)
    │   ├── metadata.json   # Training metadata and calibration temperature
    │   ├── metrics.json    # Evaluation metrics on the mixed validation set
    │   └── label_mapping.json
    └── CL_output_step2/    # After CL Round 2 (VLSP Part 2 + ViHSD rehearsal)
        ├── model/
        ├── metadata.json
        ├── metrics.json
        └── label_mapping.json
```

## How to Obtain Models

### Option 1: Load from HuggingFace Hub (recommended, automatic)
The main model (`thong7d/vihsd-xlmr-base-hate-speech`) is loaded automatically at runtime when the default environment is used:
```bash
# Default: MODEL_SOURCE=huggingface is set in .env or by the application
streamlit run streamlit_app.py
```

### Option 2: Download and cache locally
```bash
python src/data/download.py
```
This will cache the model weights into `artifacts/hate_speech_model/model/`.

### Option 3: Re-run fine-tuning
To reproduce the main fine-tuned model from scratch:
1. Open `notebooks/train_colab.ipynb` in Google Colab.
2. Mount your Google Drive and point it to the ViHSD dataset.
3. Run all cells. The output model is automatically exported to HuggingFace Hub.

### Option 4: Re-run Continual Learning
To reproduce the CL models:
1. Open `notebooks/cl_pipeline_colab.ipynb` in Google Colab.
2. Mount your Google Drive with the VLSP-2019 data at `/content/drive/MyDrive/CL_data/`.
3. Run all cells. Step 1 outputs to `CL_output_step1/`, Step 2 outputs to `CL_output_step2/`.

### Option 5: Download from Google Drive (for evaluators)
You can manually download the complete trained model directories (including CL checkpoints) from the following Google Drive link:
[Google Drive Folder - Trained Models](https://drive.google.com/drive/folders/1S6w4g_-yJaX1bynwjd5Fs3pSujZm4lJZ?usp=drive_link)
